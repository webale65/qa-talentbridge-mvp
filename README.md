# QA Testing - TalentBridge MVP

## Overview

This repository contains the QA strategy, testing documentation and quality assurance artifacts for the TalentBridge MVP project developed during the No Country professional simulation.

TalentBridge is a platform focused on improving employability for professionals aged 35-54 through skill validation, personalized learning paths and recruiter matchmaking.

---

# Current QA Stage

The project is currently in:

## QA Readiness / Pre-Testing Phase

At this stage, the QA process is focused on:

- Defining testing strategy
- Identifying business and technical risks
- Designing initial test cases
- Preparing traceability documentation
- Establishing bug reporting standards
- Organizing evidence structure
- Aligning QA with MVP business objectives

Testing execution will be performed incrementally according to feature availability delivered by the development team.

---

# QA Scope

The current MVP scope includes:

- Authentication (Register / Login)
- Candidate onboarding
- Recruiter onboarding
- Candidate profile
- AI-based diagnostic
- Skill validation
- Personalized learning path
- Recruiter marketplace dashboard
- Notifications system

---

# QA Approach

The QA strategy is business-oriented and preventive, prioritizing:

- Correct skill validation
- User experience for +35 professionals
- Marketplace usability
- Privacy and data protection
- Reliable level assignment
- Clear differentiation between declared and validated skills

---

# Testing Types Planned

- Functional Testing
- End-to-End Testing
- Integration Testing
- UI/UX Validation
- Negative Testing
- Exploratory Testing
- Regression Testing
- API Testing

---

# Main Risks Identified

- Incorrect diagnostic level assignment
- Exposure of sensitive candidate data
- Marketplace filtering inconsistencies
- Confusion between declared and validated skills
- User abandonment during onboarding or diagnostic flows
